# Test Results Directory

This directory contains artifacts generated by test and analysis runs for Module 01 spike tests.

## Contents

### Video Captures
- `spike-window.mp4` — Video capture of the spike window during test execution
- `spike-capture.mp4` — Alternative video capture from earlier analysis

### Frame Extracts
- `spike-frames/` — Directory containing individual PNG frames extracted from video
  - `frame_0000.png` — Frame 0 (active rendering)
  - `frame_0001.png` — Frame 1 (rendering end)
  - `frame_0002.png` — Frame 2 (window closing)

### Analysis Reports
- `spike-frame-analysis.txt` — Detailed per-frame pixel analysis from `analyze-spike-v2.py`
  - Background pixel counts
  - Non-background colored pixel counts
  - RGB channel detections
  - Triangle geometry detection results
- `spike-analysis.txt` — Analysis from initial `analyze-spike.py` run

## Interpreting Results

The frame analysis looks for:
- **Background pixels**: Expected clear color (dark blue-gray)
- **Colored pixels**: Triangle geometry (should have RGB colors at vertices)
- **Red/Green/Blue detection**: Specific color channel presence per frame
- **Triangle detection**: Contour analysis to identify 3-sided shapes

**Success criteria:**
- At least one frame has detected colors (✓ COLORS indicator)
- Colored pixel count in active frames ≥ 280K (indicating triangle rendered)
- No validation errors during rendering
