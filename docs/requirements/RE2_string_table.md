# RE2 — M15-03: String table (build-time i18n)

> Roadmap item: M15-03
> Depends on: module 05 (theme — for naming conventions; the table itself is standalone)
> Read `00_constitution.md` before this file.

## Purpose

A build-time codegen tool that reads a human-authored string file (e.g. `strings.en.txt`)
and emits a Zig `const` table. Application code calls `t("key")` to resolve a string at
comptime to a `[]const u8` literal. The primary locale (en) is baked into the binary at
zero runtime cost. One additional locale file (`strings.ru.txt`, `strings.de.txt`, etc.)
can be loaded at runtime.

## What to build

### String file format

A simple flat text file, one entry per line:

```
# This is a comment line (starts with #)
key=value
welcome_message=Welcome to the application
button.save=Save
button.cancel=Cancel
greeting=Hello, {name}!
```

- Lines starting with `#` are comments (ignored).
- Empty lines are ignored.
- Each line is `key=value` where key is a non-empty string without `=` and value is the rest.
- Keys are ASCII-only (a-z, A-Z, 0-9, `.`, `_`, `-`).
- Values are UTF-8 (for Cyrillic and Latin text).
- Values may contain `{placeholder}` markers for runtime substitution.

### Build-time codegen tool

A new executable `src/tools/string_table_codegen.zig` that:

1. Reads `strings.en.txt` from the `src/` directory.
2. Parses all key=value pairs.
3. Emits a Zig source file to stdout (or a named output path) containing:

```zig
// Generated by string_table_codegen. DO NOT EDIT.
const strings = struct {
    pub const welcome_message: []const u8 = "Welcome to the application";
    pub const @"button.save": []const u8 = "Save";
    pub const @"button.cancel": []const u8 = "Cancel";
    pub const greeting: []const u8 = "Hello, {name}!";

    /// Look up a string by comptime key. Returns the string literal.
    pub fn t(comptime key: []const u8) []const u8 {
        return comptime blk: {
            if (comptime std.mem.eql(u8, key, "welcome_message")) break :blk welcome_message;
            if (comptime std.mem.eql(u8, key, "button.save")) break :blk @"button.save";
            // ... one branch per key ...
            break :blk @compileError("unknown string key: " ++ key);
        };
    }
};
```

4. On failure (parse error, duplicate key), writes an error to stderr and exits code 1.

### Runtime generated file

The generated file is written to `src/strings.zig` and imported by application code:

```zig
const strings = @import("strings.zig");
const msg = strings.t("greeting");
```

### Build.zig integration

```zig
// In build.zig:
// 1. Run string_table_codegen as a build step before compilation.
// 2. The generated src/strings.zig is a build artifact;
//    build.zig ensures it is regenerated when strings.en.txt changes.

const strings_codegen = b.addExecutable(.{
    .name = "string_table_codegen",
    .root_source_file = b.path("src/tools/string_table_codegen.zig"),
});
const run_strings = b.addRunArtifact(strings_codegen);
// Input dependency: re-run when strings.en.txt changes.
run_strings.addFileArg(b.path("src/strings.en.txt"));
// Output: write generated Zig to src/strings.zig.
const strings_gen = run_strings.addOutputFileArg("strings.zig");
// ... add as input to the main build module.
```

### Runtime substitution

The `t()` function returns the raw string including `{placeholder}` markers. A separate
runtime helper performs substitution:

```zig
/// Substitute placeholders in a string. Placeholders are {name} markers.
/// Each entry in params replaces the matching {key} with the value.
/// Writes to buf; returns the written slice or null if buf is too small.
pub fn formatString(buf: []u8, template: []const u8, params: []const StringParam) ?[]const u8;
```

`StringParam`:

```zig
pub const StringParam = struct {
    key: []const u8,
    value: []const u8,
};
```

This is a thin wrapper over `std.fmt.bufPrint` with custom `{name}` bracket matching.

### Module location

```
src/tools/string_table_codegen.zig  — codegen executable
src/strings.en.txt                  — source string table (committed to repo)
src/strings.zig                     — GENERATED (not committed, in .gitignore)
src/app/locale.zig                  — formatString, StringParam helpers
build.zig                           — codegen step integration
```

## Non-goals (DO NOT implement — INV-5.4)

- **No plural forms** — `msgid`/`msgid_plural`/`msgstr[0]` is post-v1.
- **No ICU message format** — only simple `{key}` substitution.
- **No gender/inflection support** — Russian noun/adjective declension is post-v1.
- **No Gettext `.po`/`.mo` file format** — custom flat-file format only.
- **No runtime-loaded string tables beyond one extra locale** — the extra locale is a
  simple `strings.ru.txt` loaded into memory at startup; no hot-swap.
- **No auto-detection of the user's locale** — the app explicitly selects which locale strings
  to load.

## Acceptance criteria

1. `string_table_codegen` parses a 5-line `strings.en.txt` and emits valid Zig with `t()`.
2. `t("welcome_message")` returns the correct string literal at comptime.
3. `t("unknown_key")` produces a compile error (not a runtime failure).
4. `formatString(buf, "Hello, {name}!", &.{.key="name", .value="World"})` returns `"Hello, World!"`.
5. `formatString` returns `null` when buf is too small.
6. Missing `{placeholder}` in params leaves the marker as-is (e.g. `"Hello, {name}!"` with
   empty params returns `"Hello, {name}!"`).
7. Duplicate keys in `.txt` are caught as a build error.
8. `zig build` regenerates `src/strings.zig` when `src/strings.en.txt` changes.
