# R55 — M5-06: Build-time markup codegen tool

> Roadmap item: M5-06  
> Depends on: module 06 (`parse`, `NodeDesc`, `Attr`, `AttrValue`), R54 (error reporting)  
> Read `00_constitution.md` before this file.

## Purpose

Provide a `build.zig` step that runs `parse` over every `.ui` file in `src/` at build time
and emits a generated `.zig` file containing baked `NodeDesc` struct literals. The
production binary imports the generated file and never calls the parser at runtime
(INV-4.4). This keeps the binary small and eliminates a parsing cost from the hot path.

## What to build

### The codegen executable — `src/tools/ui_codegen.zig`

A standalone Zig program (not a library) that:

1. Reads one `.ui` file per argument from `argv`.
2. Calls `parse(arena, source, &diag)` (from module 06).
3. On success, writes a `.zig` file to stdout (or a named output path) containing baked
   `NodeDesc` struct literals.
4. On failure, writes the diagnostic to stderr and exits with code 1.

```zig
// src/tools/ui_codegen.zig — main entry point
pub fn main() !void {
    var gpa = std.heap.GeneralPurposeAllocator(.{}){};
    defer _ = gpa.deinit();
    const allocator = gpa.allocator();

    const args = try std.process.argsAlloc(allocator);
    defer std.process.argsFree(allocator, args);

    if (args.len < 3) {
        std.io.getStdErr().writeAll("usage: ui_codegen <input.ui> <output.zig>\n") catch {};
        std.process.exit(1);
    }

    const input_path  = args[1];
    const output_path = args[2];

    const source = try std.fs.cwd().readFileAlloc(allocator, input_path, 1024 * 1024);
    defer allocator.free(source);

    var arena = std.heap.ArenaAllocator.init(allocator);
    defer arena.deinit();

    var diag: ParseDiagnostic = undefined;
    const root = parse(arena.allocator(), source, &diag) catch |err| {
        if (err == error.ParseFailed) {
            std.io.getStdErr().writer().print(
                "{s}:{}:{}: error: {} — {s}\n",
                .{ input_path, diag.loc.line, diag.loc.column, diag.err, diag.message },
            ) catch {};
        }
        std.process.exit(1);
    };

    var out_buf = std.ArrayList(u8).init(allocator);
    defer out_buf.deinit();
    const writer = out_buf.writer();

    try emitHeader(writer, input_path);
    try emitNodeDecl(writer, root, "root_node", 0);

    const out_file = try std.fs.cwd().createFile(output_path, .{});
    defer out_file.close();
    try out_file.writeAll(out_buf.items);
}
```

### Code emission — `emitNodeDecl`

Recursively emit one `const` declaration per node, depth-first, innermost first (so each
parent's `children` slice can reference already-declared children).

```zig
fn emitNodeDecl(
    writer: anytype,
    node: NodeDesc,
    decl_name: []const u8,
    depth: u32,
) !void {
    // Emit children first (bottom-up).
    const children_names = try emitChildren(writer, node.children, decl_name, depth);

    // Emit attrs slice.
    const attrs_name = try emitAttrs(writer, node.attrs, decl_name);

    // Emit the node itself.
    try writer.print(
        \\pub const {s} = NodeDesc{{
        \\    .tag      = "{s}",
        \\    .classes  = "{s}",
        \\    .attrs    = &{s},
        \\    .children = &[_]NodeDesc{{ {s} }},
        \\}};
        \\
        ,
        .{ decl_name, node.tag, node.classes, attrs_name, children_names_concat },
    );
}
```

The exact identifier naming scheme: `root_node`, `root_node_c0` (first child), `root_node_c0_c1`
(second grandchild), etc. — depth and index form a unique suffix. No allocation for names:
use a fixed-size `[256]u8` buffer with `std.fmt.bufPrint`.

### Generated file format

For a file `src/screens/login.ui`:

```zig
// AUTO-GENERATED by ui_codegen from src/screens/login.ui — DO NOT EDIT
// Run `zig build codegen` to regenerate.

const m06 = @import("06/types.zig");
const NodeDesc  = m06.NodeDesc;
const Attr      = m06.Attr;
const AttrValue = m06.AttrValue;

pub const root_node_c0_a = [_]Attr{
    .{ .name = "text", .value = .{ .literal = "Username" } },
};
pub const root_node_c0 = NodeDesc{
    .tag      = "Text",
    .classes  = "",
    .attrs    = &root_node_c0_a,
    .children = &.{},
};
// ... more declarations ...
pub const root_node = NodeDesc{
    .tag      = "Column",
    .classes  = "p-4 gap-4",
    .attrs    = &.{},
    .children = &[_]NodeDesc{ root_node_c0 },
};
```

The output is valid Zig source that compiles without warnings. String values use
standard Zig string literals with escapes for `"`, `\`, and non-printable bytes.

### String escaping helper

```zig
fn writeEscapedString(writer: anytype, s: []const u8) !void {
    try writer.writeByte('"');
    for (s) |c| {
        switch (c) {
            '"'  => try writer.writeAll("\\\""),
            '\\'  => try writer.writeAll("\\\\"),
            '\n' => try writer.writeAll("\\n"),
            '\r' => try writer.writeAll("\\r"),
            '\t' => try writer.writeAll("\\t"),
            0x00...0x1F, 0x7F...0xFF => try writer.print("\\x{X:02}", .{c}),
            else => try writer.writeByte(c),
        }
    }
    try writer.writeByte('"');
}
```

### `build.zig` integration — `CodegenStep`

Add a `zig build codegen` step to `build.zig`:

```zig
// In build.zig:
const codegen_exe = b.addExecutable(.{
    .name   = "ui_codegen",
    .root_source_file = b.path("src/tools/ui_codegen.zig"),
    .target = b.host,
    .optimize = .Debug,
});
codegen_exe.root_module.addImport("m06", module_06);

// For each .ui file, add a RunStep.
const ui_files = &[_][]const u8{
    "src/screens/login.ui",
    "src/screens/main.ui",
    // ... enumerate at build time via std.fs.Dir.iterate in a custom step if needed
};

const codegen_step = b.step("codegen", "Regenerate baked NodeDesc files from .ui markup");
for (ui_files) |ui_path| {
    const out_path = replaceExtension(b, ui_path, ".zig");  // e.g. src/screens/login.ui → src/screens/login.ui.zig
    const run = b.addRunArtifact(codegen_exe);
    run.addArg(b.path(ui_path).getPath(b));
    run.addArg(b.path(out_path).getPath(b));
    codegen_step.dependOn(&run.step);
}
```

The `.ui.zig` output files are committed to the repository (they are baked literals, not
generated artifacts that must be gitignored). Developers run `zig build codegen` after
editing any `.ui` file. CI runs `zig build codegen` and then checks `git diff` is clean.

### Production binary — no parser import

The main app module imports the generated `login.ui.zig` directly:

```zig
// src/app/screens/login.zig
const baked = @import("login.ui.zig");

pub fn instantiateLogin(scene: *Scene, tokens: Tokens) !ElementId {
    return scene.instantiate(baked.root_node, tokens);
}
```

The module 06 `parse` function is **not** imported by the production binary (only by the
codegen tool and the hot-reload path). `build.zig` ensures module 06 is not a dependency of
the main app binary unless `-Dhot-reload=true` is set.

### Handling `{bind ...}` in generated code

`AttrValue.bind` is emitted as-is:

```zig
.value = .{ .bind = "user.name" },
```

The bind path is a string literal in the generated file. Its meaning is resolved by the
binding system at runtime (M2-04, M5-03).

### Module location

```
src/tools/ui_codegen.zig  — the codegen executable
build.zig                  — CodegenStep wiring, ui_files list

docs/requirements/R55_build_time_markup_codegen.md
```

## Public API

The codegen tool is a standalone executable; it has no library API. Its contract:

- Exit code 0 → output file written, valid Zig source.
- Exit code 1 → diagnostic to stderr, no output file written.

## Non-goals (DO NOT implement — INV-5.4)

- **No glob/auto-discovery of `.ui` files** — the list in `build.zig` is explicit (no
  `std.fs.Dir.iterate` at build time for v1; explicit enumeration is simpler and avoids
  subtle path issues).
- **No incremental codegen** — all listed `.ui` files are regenerated on each
  `zig build codegen` run; no file-level dirty detection.
- **No validation of bind paths** — the tool only parses markup structure; whether bind
  paths refer to valid signal fields is checked by the Zig compiler when the generated
  file is compiled.
- **No source maps** — no mapping from generated Zig identifiers back to source line/column.
- **No minification or compression** — output is human-readable Zig source.
- **No watch mode** — continuous file watching is M5-07 (hot-reload).
- **No embedded `.ui` source in the binary** — only the generated Zig struct literals ship;
  the original `.ui` text is a dev artifact.

## Acceptance criteria

1. `zig build codegen` runs without errors on the existing `.ui` files in `src/`.
2. The generated `.ui.zig` files compile without errors (`zig build` succeeds after
   `zig build codegen`).
3. A small round-trip test: parse a known `.ui` file, generate `.ui.zig`, import it, and
   verify the root `NodeDesc` fields match the original markup by inspection or a test that
   calls `scene.instantiate(baked.root_node, tokens)`.
4. Malformed `.ui` input causes `ui_codegen` to exit code 1 and print a diagnostic with
   line and column (using R54's `ParseDiagnostic`).
5. Special characters in string values (`"`, `\`, newlines) are correctly escaped in the
   generated output.
6. CI check: `zig build codegen` followed by `git diff --exit-code` passes (generated
   files are up to date).
7. Checklist fully ticked.

## Open questions

One: auto-discovery vs explicit list. For v1 an explicit list is fine. If the project grows
past ~20 screens, a `build.zig`-time `std.fs.Dir.iterate` scan would be cleaner. Defer
that decision until the list becomes unwieldy. Surface to the human before implementing auto-
discovery.
