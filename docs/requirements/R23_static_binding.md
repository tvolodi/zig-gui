# R23 — M2-04: Static screen data binding

> Roadmap item: M2-04  
> Depends on: M2-01 (Signal type), M2-02 (dirty scan, `refreshBindings` stub), module 07 (Scene)  
> Invariant: INV-4.1 — two binding mechanisms, each in its lane  
> Read `00_constitution.md` before this file.

## Purpose

Connect `Signal([]const u8)` fields in a typed application state struct to specific text
elements in the `Scene`, with the connection established at compile time — no runtime string
lookup (INV-4.1 "static lane"). When a signal changes, the bound element is marked dirty AND
its text is updated in the `Scene` before the next paint.

After this item ships, a static screen author writes:

```zig
const State = struct {
    greeting: Signal([]const u8),
};

var state = State{
    .greeting = try Signal([]const u8).init(gpa, "Hello", &app.scene.elements.dirty),
};
defer state.greeting.deinit();

// Bind once, after instantiate, before run.
try app.bindings.bindText(State, "greeting", &state, label_id.index, gpa);

// Later in response to any event:
state.greeting.set("World");   // marks label_id.index dirty; next frame shows "World"
```

## What to build

### `binding.zig` — `src/app/binding.zig`

```zig
const std    = @import("std");
const signal = @import("signal.zig");
const m07    = @import("../07/types.zig");

const Signal = signal.Signal;
const Scene  = m07.Scene;

/// A registered connection between one Signal([]const u8) and one element index.
pub const TextBinding = struct {
    element_idx: u32,
    /// Type-erased pointer to a live `Signal([]const u8)`.
    signal_ptr: *anyopaque,
    /// Reads the current string value from the signal. Points to a function
    /// generated by `bindText` that knows the concrete type — no vtable overhead.
    read_fn: *const fn (*const anyopaque) []const u8,
};

/// Collection of all active data bindings for a static screen.
/// Lives as a field on `App` (initialized empty in `App.init`).
pub const BindingSet = struct {
    text: std.ArrayListUnmanaged(TextBinding) = .empty,

    pub fn init() BindingSet
    pub fn deinit(self: *BindingSet, gpa: std.mem.Allocator) void

    /// Copy every bound signal's current text into the matching Scene element slot.
    /// Called once per dirty frame by App.run() before Scene.measurePass.
    pub fn refresh(self: *const BindingSet, scene: *Scene) void

    /// Bind a Signal([]const u8) field to a text element.
    ///
    /// Comptime: `StateType` is the application state struct type; `field_name`
    /// is the field name as a string literal. The compiler verifies at compile time
    /// that `StateType.field_name` is `Signal([]const u8)`.
    ///
    /// Runtime: subscribes `element_idx` to the signal (so signal.set() marks it dirty)
    /// and appends a `TextBinding` entry for `refresh()`.
    pub fn bindText(
        self: *BindingSet,
        comptime StateType: type,
        comptime field_name: []const u8,
        state: *StateType,
        element_idx: u32,
        gpa: std.mem.Allocator,
    ) !void
};
```

### `BindingSet.init` and `deinit`

```zig
pub fn init() BindingSet {
    return .{};
}

pub fn deinit(self: *BindingSet, gpa: std.mem.Allocator) void {
    self.text.deinit(gpa);
}
```

No heap allocations until the first `bindText` call.

### `BindingSet.bindText` — the comptime binding

```zig
pub fn bindText(
    self: *BindingSet,
    comptime StateType: type,
    comptime field_name: []const u8,
    state: *StateType,
    element_idx: u32,
    gpa: std.mem.Allocator,
) !void {
    // COMPILE-TIME check: the named field must be exactly Signal([]const u8).
    comptime {
        const FieldType = @TypeOf(@field(@as(*StateType, undefined).*, field_name));
        if (FieldType != Signal([]const u8)) {
            @compileError("bindText: field '" ++ field_name ++
                "' must be Signal([]const u8), got " ++ @typeName(FieldType));
        }
    }

    // RUNTIME: resolve field pointer via @field — comptime-resolved offset, no string
    // lookup at runtime (INV-4.1).
    const sig: *Signal([]const u8) = &@field(state, field_name);

    // Subscribe the element to the signal — signal.set() will dirty element_idx.
    try sig.subscribe(element_idx);

    // Generate a typed read function at comptime for zero-overhead read in refresh().
    const ReadFns = struct {
        fn read(ptr: *const anyopaque) []const u8 {
            return @as(*const Signal([]const u8), @ptrCast(@alignCast(ptr))).get();
        }
    };

    try self.text.append(gpa, .{
        .element_idx = element_idx,
        .signal_ptr  = sig,
        .read_fn     = &ReadFns.read,
    });
}
```

The `@field(state, field_name)` call is resolved by the compiler to a direct field offset.
`field_name` is a comptime constant (`[]const u8` compile-time string), so **no runtime
string comparison or hash lookup occurs** — this is exactly what INV-4.1 requires.

### `BindingSet.refresh`

```zig
pub fn refresh(self: *const BindingSet, scene: *Scene) void {
    for (self.text.items) |b| {
        const text = b.read_fn(b.signal_ptr);
        scene._text.items[b.element_idx] = text;
    }
}
```

`scene._text` is the parallel text array in `Scene` (module 07). If `_text` is not directly
accessible from `binding.zig` (same package but underscore convention), `Scene` must expose
a `pub fn setText(self: *Scene, idx: u32, text: []const u8) void` accessor. Add that method
to `Scene` rather than changing the field visibility — surface the question before
implementation if the access model is unclear.

### `App` integration — `src/app/app.zig`

`App` gains one field:

```zig
bindings: BindingSet = BindingSet.init(),
```

`App.deinit()` gains:

```zig
self.bindings.deinit(self.gpa);
```

Replace the `refreshBindings` stub added in M2-02:

```zig
// Before (stub from M2-02):
fn refreshBindings(self: *App) void { _ = self; }

// After (M2-04):
fn refreshBindings(self: *App) void {
    self.bindings.refresh(&self.scene);
}
```

`refreshBindings` is called in `App.run()` inside the dirty-check branch, before
`scene.measurePass`, exactly where the M2-02 stub was placed.

### Binding lifetime contract

- `Signal` must outlive the `BindingSet` (or at least the last `refresh()` call).
- `Scene` must outlive the `BindingSet`.
- `BindingSet` does NOT own `Signal` instances — those are owned by the application state
  struct, which is owned by the application.
- After calling `scene.reset()` (screen navigation), `BindingSet` must also be reset — the
  element indices become invalid. The mechanism for this is out of scope for M2-04 (screen
  navigation is post-v1); document in a comment.

### Module location

```

```

`binding.zig` imports: `std`, `signal.zig`, and module 07 types for `Scene`.

## Non-goals (DO NOT implement — INV-5.4)

- **No style bindings** (color, padding, etc.) — text strings only in M2-04.
- **No binding for `Computed(T)`** — `Computed([]const u8)` binding is post-M2; add
  `bindComputedText` if a later milestone requires it.
- **No runtime-path binding** — runtime string lookup into a `Value` tree is the dynamic
  lane (schema forms, module 08). INV-4.1 forbids mixing lanes.
- **No two-way binding** — `refresh()` pushes signal → scene only; scene never writes back.
- **No `bindText` for numeric signals** (`Signal(u32)`, `Signal(f32)`) — format to string
  in application code before calling `set()`; add a format helper if commonly needed.
- **No binding for non-text attributes** — layout dimensions, colors, etc. are post-v1.
- **No binding registry or named lookup** — bindings are anonymous; there is no
  `unbind(name)` or "find binding by field name at runtime."

## Acceptance criteria

1. `zig build test-binding` runs `src/app/binding_test.zig`. Tests must cover:
   - After `bindText`, `Signal.set("X")` marks `element_idx` dirty in the bitset.
   - After `refresh(scene)`, `scene._text.items[element_idx]` (or equivalent) equals the
     current signal value.
   - `deinit()` produces no leaks with `std.testing.allocator`.
   - Calling `bindText` with a field that is NOT `Signal([]const u8)` causes a **compile
     error** (verify with a `comptime` test or a note to manually confirm).
   - Calling `refresh` with multiple bindings updates all bound elements correctly.
2. `App.run()` calls `refreshBindings()` inside the dirty branch before `measurePass` —
   verified by inspection or a test double.
3. `App.deinit()` calls `bindings.deinit` — verified by leak test with
   `std.testing.allocator`.
4. Checklist fully ticked.

## Open questions

1. **`scene._text` access:** `Scene._text` (module 07) is currently named with a leading
   underscore. Before implementing `refresh()`, confirm whether `binding.zig` can write to
   it directly (same package) or whether a `Scene.setText(idx, text)` accessor is needed.
   Do NOT change `Scene`'s public type signatures unilaterally — surface if a new method
   is required.

2. **Binding invalidation on `scene.reset()`:** after navigating to a new screen and calling
   `scene.reset()`, the stored element indices in `BindingSet` are invalid. A safe pattern
   is to also call `bindings.deinit(gpa)` and rebuild bindings for the new screen. Confirm
   with the human that this is the expected usage before implementing screen navigation.
