> via `zig build test-09` (pure tests always; GPU tests on a Vulkan-capable machine) (`INV-5.2`).

## Functional — serializer (pure CPU)

- [x] Public signatures in `types.zig` match `docs/specs/09.types.zig` exactly (`INV-5.1`).
- [x] All pure CPU tests in `acceptance_test.zig` pass, unmodified (`INV-5.3`).
- [x] `buildDrawList` emits zero commands for an invisible element (transparent background,
      no border, no text).
- [x] `buildDrawList` emits zero commands for an element with `computed.w == 0` or
      `computed.h == 0`.
- [x] `buildDrawList` emits a `filled_rect` with the correct color and non-zero rect for a
      widget with a solid background (e.g. Button with default accent background).
- [x] `buildDrawList` emits a `border_rect` for a widget whose `ComputedStyle.border_width > 0`.
- [x] Commands are emitted in depth-first pre-order: parent `filled_rect` before child
      `filled_rect` (painter's algorithm).
- [x] `buildDrawList` emits `glyph` commands for a text-bearing element (font present); each
      glyph has a non-zero `dst` and a `uv` inside [0, 1].
- [x] `clampBorderWidth` clamps `border_rect.width` to `min(rect.w, rect.h) / 2`.
- [x] `expandBorderToQuads` produces four correctly positioned `FilledRect`s covering the
      top, bottom, left, and right edges of the original rect.
- [x] An empty scene → empty command list → no crash.

## Functional — GPU pipeline

- [x] All GPU tests in `acceptance_test.zig` pass on a Vulkan-capable machine (zero
      validation errors in all cases).
- [x] `VulkanBackend.initQuadPipeline` creates the quad pipeline alongside the existing
      triangle pipeline without destroying it.
- [x] `VulkanBackend.deinitQuadPipeline` destroys only the quad pipeline resources; the
      triangle pipeline, render pass, and swapchain are unaffected.
- [x] `VulkanBackend.drawFrame` with an empty command list is a no-op (no Vulkan draw call
      issued, no validation error).
- [x] `VulkanBackend.drawFrame` with a non-empty command list uploads vertices and issues
      exactly one `vkCmdDraw` call (or `vkCmdDrawIndexed` if indexed) per non-empty frame.
- [x] Alpha blending is enabled in the quad pipeline; glyph edges blend correctly against
      the background (verified visually on GPU, noted here as manual check).
- [x] `GpuAtlas.upload` creates a `VK_FORMAT_R8_UNORM` image, uploads via staging buffer,
      transitions to `SHADER_READ_ONLY_OPTIMAL`, frees the staging buffer.
- [x] `GpuAtlas.deinit` destroys the image, view, sampler, and memory without error.
- [x] Re-upload after atlas generation change: old resources freed, new ones created, no
      resource leak (confirmed by Vulkan validation layer — zero warnings).

## Architecture compliance

- [x] `buildDrawList` does not call `layout.solve` or touch `LayoutNode` directly; it only
      reads `layout.computed` (the solved rect). The caller is responsible for running solve.
- [x] The renderer (`VulkanBackend.drawFrame`) accepts only `[]DrawCommand` and `*anyopaque`
      (atlas handles); it does NOT import or reference `Scene`, `ElementStore`, or any widget
      type (`INV-2.3`). The *anyopaque cast is type-safe: the layout matches GpuAtlas.
- [x] `buildDrawList` uses a single ArrayList grown once; no per-element heap objects (`INV-3.1`).
- [x] The triangle pipeline and `drawTriangle` in module 01 are unchanged; module 01's smoke
      test still passes after module 09 is integrated.
- [x] The quad shaders (`quad.vert`, `quad.frag`) are compiled to SPIR-V by `glslc` at build
      time and `@embedFile`d — no runtime shader compilation (`INV-4.4` spirit).
- [x] `GpuAtlas` uses `*anyopaque` to store Vulkan handles, keeping Vulkan headers out of
      `09/types.zig`; Vulkan destroy calls are in `src/01/types.zig` only.
- [x] Imports only std + modules 01/02/03/05/07; nothing higher-numbered (`INV-3.4`).
- [x] No new external dependency introduced (`INV-5.6`); glslc was already approved.
