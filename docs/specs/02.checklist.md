## Right vs wrong (counter-examples — read before implementing)

**Layer separation**
- RIGHT: `wrap`/`measureWidth`/`blockHeight` take plain `Word` widths and have no font or GPU
  dependency, so they unit-test without a font.
- WRONG: make `wrap` call into `Font` directly. That couples the testable core to a C library
  and a real font, defeating the headless-logic split.

**Line breaking**
- RIGHT: break only at whitespace; an over-long word overflows its own line.
- WRONG: break mid-word to fit `max_width`, or implement the full Unicode line-break
  algorithm. Both are out of scope (`INV-1.3`); the oversized-word test guards the first.

**Atlas growth**
- RIGHT: when a glyph does not fit, grow the atlas and keep prior rects valid (repack or
  extend); re-lookup returns correct rects.
- WRONG: fail/return an error when full, or invalidate earlier glyphs on growth. The
  growth test catches the latter.

**Kerning**
- RIGHT: read the `kern` table; return 0 when absent and document that GPOS-only fonts get no
  kerning.
- WRONG: try to parse GPOS, or pull in HarfBuzz to "do it properly." Out of scope (`INV-1.3`).

**Glyph caching**
- RIGHT: rasterize each glyph once, cache in the atlas, reuse by `GlyphKey`.
- WRONG: re-rasterize on every `layoutParagraph` call — wasteful and slow.

**Scope**
- RIGHT: one font + one size per paragraph in v1.
- WRONG: build per-run styling / mixed fonts / rich text now (`INV-5.4`).
