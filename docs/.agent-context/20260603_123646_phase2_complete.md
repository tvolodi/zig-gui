## What Was Built

### Module 07 (`src/07/types.zig`)
- **5 new widget kinds:** `progress_bar`, `spinner`, `tabs`, `tab_item`, `accordion` (WidgetKind enum now has 20 members)
- **3 new state structs:** `ProgressState`, `TabsState`, `AccordionState`
- **3 new parallel arrays on Scene:** `_progress_state[]`, `_tabs_state[]`, `_accordion_state[]`
- **New Scene field:** `frame_count: u64 = 0` and `frame_time_ms: u64 = 0` (written by app.zig each frame; read by buildDrawList for animation)
- **New Scene methods:** `progressStateOf`, `setProgress`, `setIndeterminate`, `tabsStateOf`, `selectTab`, `accordionStateOf`, `toggleAccordion`, `isAccordionOpen`
- **tagToKind:** `ProgressBar`, `Spinner`, `Tabs`, `TabItem`, `Accordion` entries added
- **defaultLayoutFor / defaultStyleFor:** new cases for all 5 kinds
- **focusable_indices:** `.accordion` added
- **instantiateNode:** parallel array boilerplate + `progress_bar` attribute parsing (`value=`, `indeterminate=`); `tab_item` label→`_text`; post-recursion processing for `.tabs` (counts tab_items, hides all panels except first) and `.accordion` (finds body child, hides it)
- **deinit/reset:** all 3 new arrays handled

### Module 09 (`src/09/types.zig`)
- **buildDrawList:** signature reverted to 6 parameters (no `frame_count` — reads from `scene.frame_count` instead); this makes it compatible with `docs/specs/09.acceptance_test.zig`
- **New rendering cases:**
  - `progress_bar`: track rect + fill rect (determined fill); OR moving 40% band for indeterminate (animates via `scene.frame_count % 120`)
  - `spinner`: 8 tick marks at equal angles, rotation phase via `scene.frame_count % 8`, opacity fades by age
  - `tabs`: tab bar from `tab_item` children's `_text` labels; active tab highlighted with `tokens.accent` background
  - `tab_item`: pass-through (only renders children when not hidden)
  - `accordion`: chevron prepended to the header child using `filled_rect` (two bars at angle), chevron changes direction based on `AccordionState.open`

### App layer (`src/app/app.zig`)
- **`frame_count: u64` on AppInner:** incremented each frame; mirrored to `scene.frame_count` and `scene.frame_time_ms`
- **`hasAnimatedElements(scene)`:** scans `_kind[]` for spinners or indeterminate progress bars; returns `true` if any exist
- **Idle check updated:** calls `pollEvents()` when animated elements present (instead of `waitEvents()`), enabling frame-by-frame animation without input events
- **`handleMousePress` updated:** accordion toggle on `.accordion` click; tab bar hit-test for `.tabs` (top 36px of tabs element → compute tab_i → `selectTab`)

### `src/app/toast.zig` (new file)
- `ToastManager` with `init(overlay)`, `show(msg, kind, dur_ms, now_ms)`, `tick(...)`, `dismiss(idx)`, `deinit(alloc)`
- Max 4 simultaneous toasts. `ToastKind`: info, success, warning, error.
- Bottom-right stack with 12px gap. Each toast: background colored by kind, message text via `layoutParagraph`.
- Memory: `current_cmds: ?[]DrawCommand` slice freed at start of each tick.

### `src/app/dialog.zig` (new file)
- `DialogManager` with `init(overlay)`, `open(content_idx, scene)`, `close(scene)`, `isOpen()`, `buildOverlay(w, h, tokens, overlay, alloc)`, `deinit(alloc)`
- Semi-transparent backdrop (black, a=180) + centered white panel background (480×320).
- Focus trap: saves `prev_focused_idx` on `open`; restores on `close`.
- Memory: `current_cmds: ?[]DrawCommand` slice freed at start of each `buildOverlay`.

## No New Patterns for AGENT_GUIDE.md

The `frame_count`-on-Scene pattern (vs. parameter) is a new pattern: **When animation needs a frame counter inside a pure drawing function, add the counter as a field on the shared state struct (Scene) and have the app layer update it before calling the function. Do NOT add it as a function parameter if the acceptance test already defines that function's arity.**
