# R62 (M6-03) — Text Selection: Implementation Complete

**Date:** 2026-06-03 10:16:21

## What was built

R62 adds byte-offset text selection to both read-only `Text` elements and editable `Input`
elements. Key additions:

### Module 02 (`src/02/types.zig`)
- `PositionedGlyph` gains a `byte_offset: u32` field recording the byte position of each
  glyph's codepoint in the source string.
- `layoutParagraph` updated: a parallel `byte_offsets_buf` is maintained alongside
  `codepoints_buf` during the tokenization pass; each `PositionedGlyph` is populated with
  `.byte_offset = byte_offsets_buf.items[span.start + ci]`.
- Contract file `docs/specs/02.types.zig` updated to match.

### Module 07 (`src/07/types.zig`)
- `TextSelection` struct added (after `ScrollState`): `anchor: u32`, `active: u32`,
  `isEmpty()`, `range() → {lo, hi}`.
- `InputState.selection_start: u32` removed; selection now lives in `_selection[]`.
- `Scene._selection: std.ArrayListUnmanaged(TextSelection)` parallel array added.
- `deinit`, `reset`, `instantiateNode` all updated to manage `_selection`.
- `setInputText` updated to sync `_selection[idx]` to cursor position after setting text.
- Three new public methods: `selectionOf(idx) *TextSelection`, `setSelection(idx, anchor, active)`,
  `clearSelection(idx)`.
- Contract file `docs/specs/07.types.zig` updated to match (stubs added for all three methods).

### Module 09 (`src/09/types.zig`)
- Selection highlight block (step 2.5) inserted between step 2 (border) and step 3 (glyphs).
  Calls `text_mod.layoutParagraph` on the element's text, iterates glyphs, emits `filled_rect`
  commands for contiguous runs where `g.byte_offset` falls within `sel.range()`.
  Highlight color: `tokens.accent` with `a = 80`.
- `.input` case in `buildDrawList` cleaned up: `sel_lo`/`sel_hi` variables and old
  `selection_start`-based highlight block removed; only cursor rendering remains.

### App layer (`src/app/app.zig`)
- `AppInner.dragging_text_idx: ?u32 = null` added.
- `.mouse_move` handler updated: while `left_mouse_down` and `dragging_text_idx != null`,
  updates `scene.selectionOf(didx).active` via `hitTestText`.
- `handleMousePress` restructured: after focusing/interacting with focusable elements,
  iterates `.text` elements to start a drag selection via `hitTestText` + `setSelection`.
- `handleMouseRelease` now clears `dragging_text_idx = null` first.
- `handleKey` switch adds `.text => self.handleTextKey(focused, key, mods)`.
- `handleChar` and `handleInputKey` updated: all `inp.selection_start` references replaced
  with `scene.selectionOf(idx)` calls. `sel.anchor` and `sel.active` kept in sync with
  `inp.cursor` throughout.
- `handleTextKey` added: handles Left/Right/Home/End/Ctrl+A/Ctrl+C for focused `.text` elements.
- `hitTestText` added: lays out the text element's glyphs, finds the glyph whose horizontal
  midpoint is closest to `mouse_x`, returns its `byte_offset`.

### Documentation

- `docs/HOW_TO_USE.md`: Section 16 "Text selection (M6-03 / R62)" added.

## No new patterns recorded

R62 did not introduce a fundamentally new architectural pattern. The parallel-array extension
pattern (`ensureTotalCapacity` + `items.len = needed`) was already established by R52.
The new `hitTestText` / highlight loop are application-layer helpers, not framework primitives.
