# R64 (M6-05) — Font Fallback — Complete

R64 added a fallback font chain to `FontFamily` and wired it through `layoutParagraphEx` so that codepoints absent from the primary face are automatically served by an ordered list of up to four fallback `Font` objects. To satisfy the upward-import prohibition (INV-3.4), `FontFamily` was moved from `src/app/font_family.zig` into `src/02/types.zig`; the app-layer file is now a one-line re-export (`pub const FontFamily = @import("../02/types.zig").FontFamily;`). `GlyphKey` gained a `font_id: u8 = 0` field to distinguish primary-font atlas entries from fallback entries. `Font.glyphIndex(codepoint) i32` was added (wrapping `stbtt_FindGlyphIndex`). `REPLACEMENT_CODEPOINT: u21 = 0xFFFD` was defined. A new 7-param `layoutParagraphEx` implements the per-codepoint fallback loop; the original 6-param `layoutParagraph` is unchanged and delegates to `layoutParagraphEx(..., null)`, preserving all existing tests. Callers in `src/07/types.zig` (`measurePass`) and both text-layout calls in `src/09/types.zig` (`buildDrawList`) were updated to pass `scene.font_family`. All tests pass: `zig build test-02` (exit 0), `zig build test-07` (exit 0), and the test-09 binary (all 12 tests pass). Documentation updated: `docs/specs/02.types.zig` (all R64 stubs), `docs/specs/glossary.md` (`REPLACEMENT_CODEPOINT` entry), `docs/AGENT_GUIDE.md` (module 02, 09, and App layer notes updated; R63 entry preserved), `docs/HOW_TO_USE.md` (Step 6 section expanded with `FontFamily.addFallback` example and fallback-behavior explanation). Constitution invariant INV-1.3 was already amended on 2026-06-03 when R64 was approved; no further constitution changes required.
