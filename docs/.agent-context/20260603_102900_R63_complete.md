# R63 — M6-04 Multi-line Textarea Widget — Completion Summary

## What was built

R63 adds the `.textarea` `WidgetKind` (12th kind) to zig-gui. Users write `<Textarea>` in
markup to get a multi-line, scrollable, editable text widget. Text content is stored in the
existing `InputState.text` (`ArrayListUnmanaged(u8)`). A new parallel array
`Scene._textarea_state: ArrayListUnmanaged(TextareaState)` holds per-textarea metadata:
`line_starts` (byte offsets of each line's first character), `scroll_y`, `content_h`, and
`container_h`.

## Key implementation patterns introduced

- **Per-textarea `line_starts` management**: `line_starts` is an `ArrayListUnmanaged(u32)` embedded in `TextareaState`. It is rebuilt by `rebuildLineStarts()` after every mutation. `deinit` and `reset` free each element's `line_starts` before clearing the outer array.
- **Textarea draw as self-contained block**: Unlike `scrollview`, textarea has no child widgets. `buildDrawList` handles all rendering inline and emits `restore_scissor` directly (does not push children to the stack).
- **Cursor positioning per-line**: The cursor's byte offset is relative to the full text; `textareaLineOfByte` (binary search on `line_starts`) identifies the line, and the offset within that line is `inp.cursor - line_start`. `computeTextX` is called on the line's slice with that relative offset.
