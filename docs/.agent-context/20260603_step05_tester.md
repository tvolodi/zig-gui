## Parser colon finding

File: `docs/specs/06.types.zig`, line 262–264 (also re-exported from `src/06/types.zig` via
`pub usingnamespace`):

```zig
fn isNameChar(c: u8) bool {
    return std.ascii.isAlphanumeric(c) or c == '-' or c == '_' or c == '.';
}
```

`:` (colon, ASCII 0x3A) is NOT included in allowed name characters. As a result, any attempt
to call `markup_mod.parse` on markup containing `style:background="..."` or any other
`namespace:attr` syntax will fail with `error.UnexpectedToken` when the parser encounters
the `:` after the attribute name prefix.

This is a known limitation documented by the Test Designer. The R50 tests in
`src/07/07_test.zig` work around this by constructing `NodeDesc` structs directly rather
than parsing markup strings.

Whether this is a bug to fix or a deferred feature is a design question for the orchestrator.
The parser can handle `style:*` attributes from the `.ui` file format only if `isNameChar` is
updated to include `:`. The R50 spec says "No parser changes are needed" — but that claim
applies to runtime instantiation, not to parsing itself.

## Failures requiring implementer action

### FAIL 1 — `zig build codegen` compilation error (R55)

**Target**: `src/tools/ui_codegen.zig`

**Root cause**: `std.heap.GeneralPurposeAllocator` was removed in Zig 0.16. The same removal
affected `std.fs.cwd()` (also used on lines 35 and 64). The implementer applied the correct
Zig 0.16 fix in `src/app/file_watcher.zig` but did not apply it to `src/tools/ui_codegen.zig`.

**Lines to fix**:

1. Line 20: Replace `std.heap.GeneralPurposeAllocator(.{}){}` with the Zig 0.16 equivalent.
   In test files the codebase uses `std.heap.ArenaAllocator`. For a standalone executable,
   `std.heap.page_allocator` is the simplest drop-in, or `std.heap.c_allocator` if libc is
   linked. An `ArenaAllocator` wrapping `page_allocator` would also work.

2. Lines 35 and 64: `std.fs.cwd()` was removed. Replace with `std.Io.Dir.cwd()` (same fix
   pattern as `src/app/file_watcher.zig`) or restructure using the Zig 0.16 `std.Io` API.

**Impact**: `zig build codegen` cannot run; `src/screens/example.ui.zig` cannot be regenerated
from source. The pre-committed `example.ui.zig` is still present on disk, so `zig build`
passes, but the codegen pipeline is broken for any future regeneration.
