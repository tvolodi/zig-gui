# INV-5.1 Violation Fix — R60 Signature Restoration

**Date:** 2026-06-03  
**Task:** Fix INV-5.1 violations introduced by the R60 bold/italic implementation that changed signatures in `src/07/types.zig` and `src/09/types.zig`.

---

## What was built

R60 had changed `measurePass` and `buildDrawList` to take `*FontFamily` instead of `*Font`, and `drawFrame` to take an extra `image_atlas` parameter. This violated INV-5.1 because the frozen acceptance tests (`docs/specs/07.acceptance_test.zig` and `docs/specs/09.acceptance_test.zig`) call these with the original signatures.

**Strategy:** Store `font_family` as an optional field on `Scene`. App code sets it before calling `measurePass`/`buildDrawList`; test code leaves it null and passes a `*Font` directly.

---

## Exact changes

### `src/07/types.zig`
- Added `font_family: ?*font_family_mod.FontFamily = null` field to `Scene`
- Restored `measurePass` signature to `(self: *Scene, font: *text.Font, atlas: *text.GlyphAtlas) text.FontError!void`
- Updated body: uses `self.font_family.?.face(bold, italic)` when set, else falls back to `font` param

### `src/09/types.zig`
- Removed `font_family_mod` import (no longer directly needed)
- Restored `buildDrawList` signature: `font: *text_mod.Font` replaces `family: *font_family_mod.FontFamily`
- All 4 `family.face(...)` calls replaced with `if (scene.font_family) |fam| fam.face(...) else font`

### `src/01/types.zig`
- Restored `VulkanBackend.drawFrame` to 2 explicit params: removed `image_atlas: *const anyopaque`
- The `image_atlas` param was already unused (`_ = image_atlas; // TODO: ...`)

### `src/app/app.zig`
- All 3 `measurePass(&self.font_family, ...)` calls updated:
  - Add `self.scene.font_family = &self.font_family;` before each call
  - Changed to `measurePass(self.font_family.face(false, false), ...)`
- `buildDrawList` call: `&self.font_family` → `self.font_family.face(false, false)`
- `drawFrame` call: removed `&self.gpu_image_atlas` argument

---

## Tests passing

- `zig build test-07` → exit 0, all tests pass (including FONT: measurePass test)
- `zig build test-09` → all 12 tests pass (3 GPU, 9 CPU)
- `zig build test-02` → exit 0, no regressions
- `zig build` → no compile errors

## Bold/italic face selection

Still works via `scene.font_family`. App sets `self.scene.font_family = &self.font_family` before each `measurePass` and `buildDrawList` call. Elements with `font_bold=true` or `font_italic=true` in their `ComputedStyle` get the correct face from `FontFamily.face(bold, italic)` with fallback to `regular`.
